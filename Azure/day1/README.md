# ☁️ 1. What is Cloud Computing?

Cloud computing means using **servers, storage, networking, and software over the internet** instead of owning physical infrastructure.

### 🔹 Simple Explanation:
It is like electricity — **use when needed and pay only for usage**.

### 🔹 Key Points:
- No hardware ownership
- Accessible from anywhere
- Highly scalable

---

# ❗ 2. Problems Solved by Cloud

### 🔻 Before Cloud:
- High infrastructure cost  
- Slow setup time  
- Maintenance complexity  
- Poor demand prediction  

### ✅ With Cloud:
- No upfront investment  
- Instant resource provisioning  
- No maintenance  
- Easy scaling  

---

# 📜 3. History of Cloud Computing

- **1961** – Concept introduced  
- **1999** – Salesforce launched cloud apps  
- **2002–2006** – AWS introduced cloud services  
- **2009** – Google & Microsoft Azure entered  

👉 Cloud is now a core part of IT industry  

---

# ☁️ 4. Types of Cloud

## 🔹 Public Cloud
- Shared infrastructure  
- Example: AWS, Azure  
- Cost-effective  

## 🔹 Private Cloud
- Dedicated environment  
- High security  
- Expensive  

## 🔹 Hybrid Cloud
- Combination of both  
- Flexible and scalable  

---

# ⚙️ 5. Core Cloud Principles

- **On-Demand Self Service** → Instant resource creation  
- **Elasticity** → Auto scaling based on demand  
- **Pay-as-you-go** → Pay only for usage  
- **High Availability** → Systems designed to handle failure  

---

# 🧱 6. Cloud Service Models

## 🔹 IaaS
- Rent infrastructure  
- You manage OS and apps  

## 🔹 PaaS
- Deploy code directly  
- Cloud manages infrastructure  

## 🔹 SaaS
- Use software directly  
- No setup required  

---

# 🏢 7. Why Enterprises Choose Azure

- Works with Windows & Active Directory  
- Strong hybrid cloud support  
- High security & compliance  
- Built-in DevOps tools

# 🌍 8. Azure Global Infrastructure

## 🔹 Region
- Geographic location of data centers  
- Affects latency, cost, availability  

## 🔹 Region Pair
- Two regions connected for disaster recovery  

## 🔹 Availability Zones
- Multiple data centers in one region  
- Provides high availability  

---

# 🧩 9. Azure Resource Hierarchy


Tenant → Management Group → Subscription → Resource Group → Resource



## 🔹 Explanation:
- **Tenant** → Organization  
- **Subscription** → Billing unit  
- **Resource Group** → Logical container  
- **Resource** → Actual services  

### ⚠️ Important:
Deleting a resource group deletes all resources inside it.

---



## 🔹 Learning:

* Resource groups are mandatory
* Helps in organizing cloud resources

# ⚔️ 11. AWS vs Azure DevOps Services Comparison

## 🔹 CI/CD Tools

* AWS → CodePipeline, CodeBuild, CodeDeploy
* Azure → Azure DevOps (all-in-one platform)

## 🔹 Compute

* AWS → EC2
* Azure → Virtual Machines

## 🔹 Containers

* AWS → ECS, EKS
* Azure → AKS

## 🔹 Storage

* AWS → S3
* Azure → Blob Storage

## 🔹 Infrastructure as Code

* AWS → CloudFormation
* Azure → ARM Templates, Bicep

## 🔹 Security

* AWS → IAM, Secrets Manager
* Azure → Azure AD, Key Vault

## 🔹 Monitoring

* AWS → CloudWatch
* Azure → Azure Monitor

## 🔹 Networking

* AWS → VPC
* Azure → Virtual Network

## 🔹 Serverless

* AWS → Lambda
* Azure → Functions

---
